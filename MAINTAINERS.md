## Responsibilities

Lead maintainer responsibilities:

- **Code Review** — Review and approve pull requests
- **Issue Triage** — Categorize, prioritize, and respond to issues
- **Release Management** — Manage versioning, changelog, and npm publishing
- **Security** — Handle vulnerability disclosures (see [SECURITY.md](./SECURITY.md))
- **Community** — Foster a welcoming environment per [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)

## Decision Making

- Major decisions are documented in the `docs/` folder as Architecture Decision Records (ADRs)
- Changes to public API require approval from the lead maintainer
- Breaking changes follow the process in [CONTRIBUTING.md](./CONTRIBUTING.md)

## Becoming a Maintainer

 Exceptional contributors who consistently:
- Submit high-quality code and documentation
- Participate in code reviews
- Respond to and resolve issues
- Follow project conventions

 may be invited to become maintainers. There is no fixed timeline for this invitation.

## Contact

For maintainer-related matters, open an issue with the label `maintainer` or reach out via GitHub Discussions.
